What is Puppet?

Puppet is more than configuration management

Puppet manages the state of machines

Why should I use Puppet?

Puppet can guarantee the state of the machine

All changes are logged

Individual machines don't have to be configured

How does it work?

Puppet workflow

  1. Manifest compiles on the master
  2. Manifest is compared to the state of the system
  3. Changes are made
  4. A report is sent back to the puppetmaster
  5. Reports are stored on the master

Where's the code?

Puppet keeps code in modules

Modules include code to install and configure services

Secrets can be kept in Hiera to keep them separate from modules

Module infrastructure

Public module repository at forge.puppetlabs.com

Secrets are kept separate of modules to make them easy to update and share

Custom modules are easy to write and implement

About the Forge

The forge has thousands (3629 as of writing) of modules

You can probably find one that works for you
